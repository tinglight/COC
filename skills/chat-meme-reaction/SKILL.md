# 群聊表情包反应

## 概览

使用这个 Skill，让 QQ CoC bot 在群聊气氛合适时生成一张轻量反应表情包。目标不是画一张完整插图，而是像群友顺手丢出的梗图：主体少、表情准、文字短、读得快、不会打断跑团。

这个 Skill 服务 `AI_CHAT_IMAGE_PROMPT`、显式“来个表情包”请求，以及后续可能的自动氛围判断。默认生成原创贴纸，不抓取外部图片，不复刻真人或商业角色模板。

## 快速工作流

1. 判定是否该发。
   - 用户明确要表情包且内容安全时，可以发。
   - 非明确请求时，只在笑点、名场面、骰点结果、群友自嘲、bot 自嘲、等待间隙或轻松吐槽已经成型时发。
   - 严肃剧情、规则争议、真实情绪低落、隐私话题、KP-only 信息附近不要发。
2. 选一个信息来源。
   - 优先用最新群聊公开文本或 bot 刚回复的关键词。
   - 其次用本团反复出现的经典语录。
   - 再其次用通用网络语气梗；不确定是否过时时，改成低风险原创短句。
3. 选一个表情包结构。
   - 单主体反应贴纸：最稳，适合 80% 场景。
   - 大字吐槽图：适合骰点惨案、规则误会、群友金句。
   - 两格小反差：只在信息足够简单时用，避免缩小后读不清。
4. 写图片提示词。
   - 明确“compact square reaction meme/sticker, not a full illustration”。
   - 要求 1 个原创吉祥物或简化表情主体，纯色/透明/浅色背景。
   - 若需要文字，给出准确中文短字，要求大号、黑白描边、高对比、1-2 行。
5. 做安全检查。
   - 不泄露模组真相、暗线、未公开线索或 KP 私密裁定。
   - 不嘲笑单个群友的失误、外貌、身份、疾病、经济状况或真实压力。
   - 不使用真实玩家照片、真人肖像、商业 IP 角色或平台水印。

## 触发判定

需要细化“氛围到了没有”时，阅读 `references/trigger-rules.md`。核心判定：

```text
显式要图 > 群体共同笑点 > 桌边名场面 > 低风险等待间隙 > 单人尴尬/严肃剧情
```

只有前三类通常值得发。自动触发应宁可少发，不要把每次 AI 回复都配图。

## 画面规则

需要构造具体风格、文案和提示词时，阅读 `references/style-playbook.md`。默认规格：

- 正方形，适合 QQ 聊天流，小而清楚。
- 一个中心主体，表情夸张，动作简单。
- 背景极简，不做完整场景、不做电影海报。
- 中文文字 2-12 字最佳，最多 18 字；一眼读完。
- 文字是表情包的一部分，不是聊天气泡、UI、Logo 或水印。

## 文案规则

优先级：

1. **群内经典语录**：只使用公开聊过、大家明显认可为玩笑的短句。
2. **跑团桌边梗**：围绕侦查、灵感、SAN、大成功、大失败、KP 沉默、角色卡、线索整理。
3. **网络热梗语气**：只借用语气和句式，例如装死、已老实、破防、绷不住、包的、尊嘟假嘟；不要硬塞与当前群聊无关的流行词。
4. **原创短吐槽**：当热梗不贴脸时，用原创短句更自然。

不要为了显得懂梗而编造“群友经典语录”。没有把握时选原创短句。

文案必须表达一个可复用的反应、情绪或状态，不能只是描述当前任务动作。生成前做一次“这是什么梗”自检：如果解释只能是“因为我正在给用户看样图/正在生成图片/正在等待确认”，这不是表情包梗，应改成能复用的反应句。比如本轮“给用户看样图被要求修改”不应写 `先看样图`，应写 `在改了`、`已老实`、`这就返工` 或 `包能改的`。

## 输出格式

如果用户要一个可接入运行时的判断结果，输出：

```json
{
  "decision": "send",
  "reason": "最新消息形成了群体共同笑点，且不是严肃剧情或规则争议。",
  "caption": "先别SAN",
  "prompt": "Generate one compact square QQ reaction meme sticker, not a full illustration..."
}
```

如果用户只要 `AI_CHAT_IMAGE_PROMPT` 插入段，直接给可复制的提示词，不输出长篇解释。
