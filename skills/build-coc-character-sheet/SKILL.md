# CoC 角色卡建卡引导

## 目标

引导成员用短轮次完成 CoC7 调查员角色卡：先定概念和桌面要求，再处理属性、派生值、职业技能、兴趣技能、背景钩子，最后整理成本项目可保存的 `.st` 命令。

不要一次倾倒完整规则书。每轮只推进当前步骤，并在需要选择或骰点时停下来等成员或 KP 确认。

## 对话节奏

- 先判断说话者身份。KP 可以讨论模组适配和队伍结构；PL 只给无剧透建议，不透露 keeper-only 真相、结局、暗线或其他玩家私密内容。
- 如果用户只问“怎么建卡”，先给 6-8 步总览，再问“要不要从第 0 步开始”。
- 如果用户说“带我车卡/建卡”，直接从第 0 步开始，一次问 2-4 个最关键问题。
- 优先保留玩家幻想，再建议适配 CoC 和当前团的版本。不要把优化强度放在角色体验前面。
- 不替玩家伪造骰点、检定结果或 KP 裁定。需要随机属性时，引导玩家用本地 `.r` 命令或让 KP 提供房规数组。
- 默认中文、短句、适合 QQ 群阅读。表格只在总结角色卡或保存命令时使用。

## 逐步流程

### 0. 确认桌面约束

询问或确认：

- 规则版本和建卡方式：随机骰、标准数组、点数分配、KP 给定模板或房规。
- 时代、地点、团风格：调查、社交、动作、生存、怪奇、情感戏、喜剧调剂等。
- 玩家身份和边界：新手/熟手、扮演舒适度、避雷主题、角色死亡接受度。
- 本 bot 存储目标：是否只保存常用检定值，还是先完整整理再拆成 `.st`。

如果缺少模组信息，不要编造。用通用 CoC 调查员建议，并提醒以 KP 招募说明为准。

### 1. 角色概念

把角色先压成一句话：

```text
我想演一个【职业/身份】，因为【动机】卷入调查；TA 的亮点是【可扮演矛盾或弱点】。
```

给 2-4 个方向时，每个方向包含：

- 概念 pitch
- 适合的桌边体验
- 可用职业或社会身份
- 可能高光场景
- 1 个可玩的缺点
- 1 个能让 KP 拉进故事的钩子

### 2. 属性

CoC7 常用属性包括 STR、CON、SIZ、DEX、APP、INT、POW、EDU，常见还会记录 Luck。

如果玩家要随机，提示他们按 KP 认可方式掷骰，并把结果贴回来。常见方式是：

- STR、CON、DEX、APP、POW：`3d6 * 5`
- SIZ、INT、EDU：`(2d6 + 6) * 5`
- Luck：`3d6 * 5`

不要自行生成数值。玩家给出数值后，检查是否在 0-100 区间内，并帮他们解释每个高低值对扮演和技能分配的影响。

### 3. 派生值

玩家给出属性后，整理常用派生项：

- HP：通常取 `(CON + SIZ) / 10` 向下取整。
- MP：通常取 `POW / 5` 向下取整。
- SAN：初始通常等于 POW，除非 KP 或职业规则另有调整。
- Move、DB、Build：按 KP 使用的规则表确认；不确定时标为“待 KP 确认”。

本项目最常写入本地角色卡的是 `HP`、`SAN` 和常用技能。其他派生项可以先写在文字总结里。

### 4. 职业和职业技能

帮助玩家选职业时，先匹配角色概念和模组前提，再考虑技能覆盖。避免说“必须选最强职业”。

询问：

- 这个角色靠什么谋生或行动？
- TA 为什么会进入调查？
- TA 在队伍里想承担哪类功能：发现线索、社交突破、医学救援、战斗自保、知识研究、潜入、驾驶/机械、神秘学或后勤？

给职业技能建议时，区分“推荐”与“KP/规则书要求”。如果没有职业表原文，只给合理候选，不声称是官方职业包。

### 5. 技能分配

先列核心技能，再分配数值：

- 线索发现：侦查、聆听、图书馆使用。
- 社交：说服、魅惑、话术、恐吓、信用评级。
- 知识：历史、神秘学、博物学、心理学、外语、母语、法律、会计等。
- 行动：潜行、闪避、格斗、射击、驾驶、机械维修、电气维修、急救、医学。

原则：

- 每名调查员至少有 2-4 个明显强项。
- 给 1-2 个中等自保或补位技能，不要全堆单一方向。
- 保留缺点；低技能也是扮演材料。
- 新手建卡时优先让常用技能清楚可用，而不是追求全技能覆盖。

### 6. 背景与关系

把数值变成能上桌的人：

- 外貌与第一印象
- 信念/原则
- 重要之人或关系
- 意义重大的地点
- 珍视物品
- 怪癖、缺点或创伤按钮
- 调查动机
- 与其他 PC 或 NPC 的公开关系

如果涉及模组，PL 版本只写玩家可见动机；私密钩子应请 KP 单独确认。

### 7. 保存到本项目角色卡

本 bot 的本地角色卡是键值形式。把最终常用项整理成 `.st` 命令，例如：

```text
.st HP12 SAN60 侦查55 聆听45 图书馆使用60 说服50 心理学40 闪避35
```

如果内容很长，拆成多条 `.st`。保存后提示：

- 用 `.show` 查看角色卡。
- 用 `.ra 侦查` 调用已保存技能检定。
- 用 `.sc 0/1d6` 使用角色卡 SAN 并写回损失。
- 用 `.hp -3 原因` 或 `.san +2 原因` 记录后续变化。

不要把无法数值化的背景强行塞进 `.st`。建议用 `.记住` 保存关键设定和使用时机，例如：

```text
.记住 角色是战后急诊医生，害怕密闭车厢；用在：医疗、创伤或列车场景时
```

## 输出模板

阶段性总结使用这个形状：

```text
当前进度：第 X 步 / 7
已确定：
- ...

待确认：
- ...

下一步：
请在 A/B/C 中选一个，或直接告诉我你的想法。
```

最终交付使用这个形状：

```text
角色一句话：
...

核心数值：
...

上桌钩子：
...

建议保存命令：
.st ...

可选记忆：
.记住 ...
```
